Activation

Bind a key to a single machine

SP-XXXX-XXXX-XXXX-XXXX format with checksum. Server hashes on store, verifies on activation, returns a 14-day Ed25519-signed JWT bound to the device fingerprint.

SP-A2BC-K9PQ-7XHM-RTYW
Seats

Hard-cap with kick

Per-license seat limit. Idempotent re-activation from the same fingerprint. Admin can kick a seat to free it for someone else.

Heartbeat

6-hour soft expiry

Token slides forward 14 days on every heartbeat. Offline grace until soft expiry; then re-activate.

Updates

Tauri manifest endpoint

Drop-in feed for the Tauri 2 updater. Stable + beta channels, critical-priority flag, signed bundles.

Audit

Append-only log

Every activation, heartbeat, revoke, and admin action recorded with timestamp and actor. No deletions.

Admin UI

Issue, monitor, revoke

Single page per license: seats, heartbeats, audit trail, kick / revoke actions. Bcrypt-hashed admins, session cookies.

Crypto

Ed25519, not symmetric secrets

Server signs with the private key; desktop client verifies with the embedded public key. A leaked server can't issue tokens for a different signer.

Why this exists

SaaS license servers cost more than the licenses they're protecting.

Keygen starts at $79/mo for 50 keys. Cryptolens at $35/mo. LemonSqueezy takes 5% of every transaction. None let you self-host without upgrading to enterprise pricing. This is the answer when you're ready to own your activation flow — built first for SignPortal Desktop, generic enough for any Tauri or Electron client.

How it works

Three layers, one trust boundary at the JWT.

The desktop client never sees plaintext private keys. The license server never sees a user's machine fingerprint until activation. Cryptographic signature is the only thing that crosses between them.

Layer 1 — Desktop client (Tauri 2)

Activation gate locks until a valid key is entered

Computes a SHA-256 fingerprint of the machine UID. Sends activation_key + fingerprint + platform to the server. Stores the returned JWT in the OS keychain (macOS Keychain / Windows DPAPI / Linux libsecret). Verifies signatures with the embedded Ed25519 public key.

↓ POST /api/licenses/activate ↓
Layer 2 — License server (this project)

Validates key, claims seat, signs JWT

Hashes the key, looks up the license, checks revocation + expiry + seat count. Records the seat with the machine fingerprint. Issues a 14-day Ed25519 JWT with the license id, plan, fingerprint, and license expiry baked in. Logs the activation to the audit table.

↓ Returns { token: "eyJhbGciOi..." } ↓
Layer 3 — Tauri 2 client (verification)

Verifies token, unlocks the app

Verifies the JWT signature with the embedded pubkey. Confirms the fingerprint claim matches this device. Caches the token. Calls heartbeat every 6 hours to refresh; falls back to a 14-day grace if the server is unreachable.

REST API

Six public endpoints. Six admin endpoints.

All JSON. Bcrypt-hashed admin sessions. Ed25519-signed activation JWTs. SQLite-backed.

POST/api/licenses/activateClaim a seat, return a 14-day JWT bound to fingerprint.
POST/api/licenses/heartbeatRefresh the token, surface remote revocation.
POST/api/licenses/deactivateFree a seat. Used at uninstall or when moving machines.
GET/api/updates/desktop/{target}/{current}Tauri 2 updater feed. 204 if up to date, signed manifest otherwise.
POST/api/admin/loginBcrypt verify, set httpOnly cookie session.
POST/api/admin/licensesIssue a new license. Returns the raw key — once.
PATCH/api/admin/licenses/{id}Revoke, change seat limit, edit notes, extend expiry.
DELETE/api/admin/licenses/{id}/seatsKick a specific seat by fingerprint. User can re-activate.
GET/api/admin/auditFull audit log. Last 200 events with actor + details.
Quickstart

Three commands to a running license server.

SQLite ships in the binary. Ed25519 keypair generates on first run. Admin user seeds via prompt or env.

# 1. Install
git clone https://github.com/sinhaankur/cuddly-journey.git
cd cuddly-journey
npm install

# 2. Initialise database, generate signing keys, seed an admin user
npm run db:init
# Prompts for admin email + password. Prints the public key —
# paste it into the Tauri client's SIGNPORTAL_LICENSE_PUBKEY_PEM build secret.

# 3. Run
npm run dev   # http://localhost:3000 — admin UI

# Production: docker build, deploy to Render / Fly / your own box.
# Single Next.js standalone build, SQLite WAL, ~80MB image.
